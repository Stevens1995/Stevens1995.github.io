ES6中的Promise和异步编程

Promise是除了事件和回调函数之外，异步编程的另一种选择。

异步编程背景
javascript引擎是基于单线程和事件循环来构建的，一次只能执行一段代码，所以需要来监听需要执行的代码，这些代码保存在job queue中，当一段代码准备要执行时就会添加到job queue中，event loop是一段监听job queue的进程，当主程序中的代码执行完毕时，event loop检查job queue是否还有代码需要执行。

Event Model
当用户点击鼠标或者按下键盘时，触发一个事件，事件相对应的处理程序便会添加到job queue中，这是javascript中最基础的异步编程方式。在这种模式下，代码必须跟踪处理程序所对应的对象，当情况很复杂的时候，跟踪对象也变得愈发艰难。

Promise基础
一个Promise中保存着对应的异步操作的结果。
Promise的生命周期
一个Promise的生命周期中包含了三个状态：
Pending:代表异步操作还没有完成。
fullfilled：异步操作已经正确的执行。
rejected:异步操作发生错误。
Promise中有一个叫做[[PromiseState]]的内部属性，这个属性会根据Promise的状态被设置为Pending、fullfilled、rejected，这个属性并没有对外暴露出来，所以你不能手动的指定Promise的状态，但是当Promise的状态改变时，你可以通过then()方法来执行某些操作。

then()方法接受两个参数，第一个参数是一个当异步操作完成时需要执行的函数，第二个参数是当异步操作失败时需要执行的函数。这两个函数都可以向后续的操作传递一些数据，通过向这两个函数中传递参数。

创建状态不确定的Promise
通过Promise构造函数来创建一个新的Promise，Promise构造函数接受一个参数，这个参数是一个叫做executor的函数，里面包含着初始化Promise的代码，executor接受两个参数，resolve()函数和reject()函数，resolve()函数将Promise的状态转化为fullfilled，reject()函数将Promise的状态转化为rejected。

Global Promise Rejection Handling

Promise中一个令人争议的地方是，当rejected发生时，如果没有指定处理程序，failure会安静地发生而不会被人们捕捉。

当Promise中的错误发生时，处理程序并不会马上执行，而是要等到后面。

Node.js中的Rejection Handling
Node.js中的process对象有两个事件与Rejection Handling相关：
unhandledRejection:当一个Promise的状态变为rejected，如果在一轮Event loop中没有指定处理程序运行，触发这个事件。
rejectionHandled: 当一个Promise状态变为rejected，如果在一轮event loop后有指定的处理程序运行，这个事件触发。

浏览器中的Rejection Handling
浏览器中也提供两个用来处理rejection的事件，和Node.js中的一样，也是unhandleRejection和rejectionHandled

Promise链
then()和catch()方法的返回值是一个新的Promise，只有当上一个Promise的状态能够转化为fullfilled或者rejected，新的Promise才会resolved（也就是新的Promise状态才会转化为fullfilled）。

对多个Promise进行响应

Promise.all() 方法：
Promise方法接受一个可迭代的Promise集合，返回一个Promise，当集合中的每个Promise状态都变为fullfilled时，返回的Promise状态才会是fullfilled，当其中一个Promise变为rejected时，返回结果也会立刻变为rejected。

Promise.race()方法：
Promise.race()方法和Promise.all()方法一样，也是接收一个可迭代的Promise集合，返回一个Promise。当时当集合中的有一个Promise变为fullfilled时，返回的Promise的状态也立刻变为fullfilled。
